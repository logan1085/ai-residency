# Mastery Rubric

Use this rubric to track whether a module is merely visited or actually mastered.

| Level | Meaning | Evidence |
| --- | --- | --- |
| 0. Not Started | No meaningful work yet. | Module remains planned. |
| 1. Oriented | You can describe the module goals and core vocabulary. | Notes, glossary, reading list. |
| 2. Implemented | You built the central artifact with help. | Code, notebook, demo, or worked examples. |
| 3. Explained | You can explain the artifact without reading the source. | Written walkthrough, diagram, recorded explanation, or teaching note. |
| 4. Evaluated | You tested the artifact and found limitations. | Tests, eval table, failure analysis, ablations. |
| 5. Portfolio Ready | The artifact is polished enough for a reviewer. | Public page, reproducible commands, clear claims, limitations, and next steps. |
