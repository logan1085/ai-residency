# Learning System

The curriculum should feel like a residency, not a static reading list. The working model is a two-agent loop:

- **Residency Coach:** a conversational study partner you interact with during learning sessions.
- **Curriculum Maintainer:** this repo-focused Codex thread, which turns session reports into durable pages, logs, checklists, tests, and commits.

## How A Session Works

1. Start with the [Day 1 agenda](../progress/day-01.md) or the current module page.
2. Talk with the Residency Coach while you read, code, and debug.
3. End the session with a structured report.
4. Bring that report back to the Curriculum Maintainer.
5. The maintainer updates the progress dashboard, work log, module page, and next agenda.

## Why This Split Helps

The coach can stay close to your thinking: explanations, Socratic questions, debugging, confidence checks, and encouragement. The maintainer can stay close to the artifact trail: what changed, what evidence exists, what belongs in the public portfolio, and what should be committed.

## Default Learning Loop

| Phase | Question | Artifact |
| --- | --- | --- |
| Orient | What am I trying to understand or build? | Session goal |
| Attempt | Can I do it before looking at the answer? | Scratch work |
| Compare | Where does my version differ from the reference? | Diff notes |
| Explain | Can I teach it back simply? | Written explanation |
| Prove | What evidence shows progress? | Test, demo, note, diagram |
| Report | What should the repo remember? | Session report |
