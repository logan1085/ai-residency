# Module 7: Alignment and Safety

Module 7 develops the habit of making model behavior legible, bounded, and accountable.

## Learning Objectives

- Identify risks introduced by model capability, context, tools, and deployment surface.
- Design safety policies that are testable.
- Build red-team suites for realistic misuse and failure modes.
- Communicate residual risk clearly.

## Subtopics

- Preference learning and instruction following.
- Harmlessness, honesty, privacy, and autonomy boundaries.
- Red teaming and adversarial prompting.
- Safety classifiers and refusal behavior.
- Tool-use risk and permissioning.
- Risk registers and incident review.

## Assignments

- Write a safety policy for one system built earlier.
- Create a red-team set with benign, borderline, and disallowed requests.
- Evaluate false positives and false negatives.
- Document mitigations and residual risks.

## Projects

- Add a safety layer to the agent workflow.
- Build a red-team report for the RAG assistant.
- Compare two refusal or routing strategies.

## Proof of Mastery

- Safety policy with concrete examples.
- Red-team dataset and results.
- Risk register with severity and likelihood.
- Post-mitigation evaluation summary.
