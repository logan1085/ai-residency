# Module 9: Systems and Deployment

Module 9 covers the practical work required to move AI prototypes into reliable services.

## Learning Objectives

- Package an AI system behind a stable interface.
- Measure latency, throughput, cost, and reliability.
- Add monitoring and operational runbooks.
- Make deployment choices based on constraints.

## Subtopics

- API design and model serving.
- Batch versus streaming inference.
- Caching and rate limits.
- Queues, retries, and fallbacks.
- Cost modeling and capacity planning.
- Monitoring, alerts, and runbooks.

## Assignments

- Deploy one prior module project locally or to a hosted environment.
- Add structured logs for requests and model calls.
- Run a small load test.
- Write an operations note for common failures.

## Projects

- Package the RAG assistant as a service.
- Add caching and compare cost or latency.
- Build a release checklist for model updates.

## Proof of Mastery

- Deployed service or reproducible local deployment.
- Latency, throughput, and cost report.
- Monitoring screenshot or trace sample.
- Runbook for failures and rollback.
