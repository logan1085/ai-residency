# Module 4: Agents and Tool Use

Module 4 studies AI systems that take actions through explicit tool contracts, planning loops, and stateful workflows.

## Learning Objectives

- Distinguish ordinary prompting from tool-using agent design.
- Define safe, typed tool interfaces.
- Build planning, execution, observation, and recovery loops.
- Inspect traces to understand agent behavior.

## Subtopics

- Tool schemas and input validation.
- Planner-executor patterns.
- Memory, scratchpads, and state.
- Human approval boundaries.
- Error recovery and retry policies.
- Trace review and behavior debugging.

## Assignments

- Design three tools for a research assistant agent.
- Write tests for tool input validation.
- Run the agent on five tasks and classify failures.
- Add a trace log that captures plan, action, observation, and final answer.

## Projects

- Build a literature triage agent for paper abstracts.
- Build a repository maintenance agent that opens structured issues.
- Add a human-in-the-loop checkpoint for risky actions.

## Proof of Mastery

- Tool schema documentation.
- Working agent demo with traces.
- Failure taxonomy and mitigation plan.
- Clear policy for actions requiring approval.
