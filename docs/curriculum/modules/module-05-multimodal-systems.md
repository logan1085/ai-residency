# Module 5: Multimodal Systems

Module 5 expands beyond text into systems that reason over images, audio, diagrams, and mixed media.

## Learning Objectives

- Explain how multimodal encoders connect non-text inputs to language models.
- Build a pipeline that accepts at least one non-text modality.
- Evaluate multimodal outputs with task-specific examples.
- Design interfaces that expose uncertainty and evidence.

## Subtopics

- Vision-language and audio-language model families.
- Image preprocessing and prompt construction.
- OCR, diagram understanding, and document parsing.
- Captioning, classification, and grounded QA.
- Multimodal failure modes.
- Accessibility and user experience.

## Assignments

- Create a 25-example multimodal evaluation set.
- Compare zero-shot and few-shot prompting on the same task.
- Document at least five failure modes with examples.
- Add a confidence or abstention convention to the output.

## Projects

- Build a diagram explainer for AI architecture images.
- Build a document reviewer that uses text plus page images.
- Build an audio-note summarizer for research meetings.

## Proof of Mastery

- Working multimodal demo.
- Evaluation set with inputs, expected behavior, and observed behavior.
- Error taxonomy with concrete examples.
- Design note explaining where the system should not be trusted.
